# Sprints

This folder contains sprint planning documents, current sprint status, and retrospectives.

## What Goes Here

- Sprint planning documents
- Current sprint backlog and status
- Daily standup notes (if needed)
- Sprint retrospectives
- Velocity tracking

## Document Naming Convention

Use sprint numbers or dates:
- `sprint-01-planning.md`
- `sprint-01-status.md`
- `sprint-01-retrospective.md`
- `current-sprint.md` (symlink or current sprint status)

## Workflow

1. **Sprint Planning:** Create planning document with goals and tasks
2. **During Sprint:** Update status document regularly
3. **Sprint End:** Create retrospective document with learnings
4. **Archive:** Move completed sprint docs to archive/ subfolder if needed
