# Requirements

This folder contains requirement specifications, user stories, and acceptance criteria.

## What Goes Here

- Feature requirements
- User stories
- Acceptance criteria
- Functional specifications
- Non-functional requirements (performance, security, etc.)
- Business rules

## Document Naming Convention

Use descriptive names that indicate the feature:
- `feature-name-requirements.md`
- `user-stories-feature-name.md`
- `acceptance-criteria-feature-name.md`
